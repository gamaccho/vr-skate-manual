Step 1 / 5

🌅 作業開始：最新のコードを取得する

作業を始める前に、必ず main ブランチを最新の状態にします。他のメンバーが加えた変更を自分のPCに反映させる作業です。

💡 なぜ毎回やるの？ チームで作業していると、自分が作業していない間に他のメンバーがコードを更新していることがあります。古いコードのまま作業するとコンフリクト（競合）の原因になるため、必ず最初にこの手順を行います。

VS Codeのターミナルを開く

Cmd+`（Windowsは Ctrl+`）でターミナルを開く

mainブランチに移動して最新を取得

# mainブランチに移動する
git checkout main

# GitHubから最新のコードを取得する
git pull origin main
ℹ️ Already up to date. と表示された場合 すでに最新の状態です。そのまま次のステップに進んでOKです。
Step 2 / 5

🌿 作業ブランチを作る

💡 「ブランチ」って何？ プロジェクトのコードを「コピーして別の場所で作業する」イメージです。チームで共有している main（本番コード）を直接触らず、自分専用の作業スペース（ブランチ）を作ってそこで編集します。
ℹ️ ブランチは毎回使い捨て 作業のたびに新しいブランチを作ります。前回のブランチは使い回しません。名前は作業内容がわかるものにしましょう。

例：feature/enemy-ai（敵AIを追加）、fix/jump-bug（ジャンプのバグ修正）

ブランチを作成して移動する

# ブランチを作成して、そのブランチに移動する
git checkout -b feature/ブランチ名

例えば「スコアボードを追加する」作業なら：

git checkout -b feature/scoreboard
✅ 確認方法 git branch を実行すると現在いるブランチが * で表示されます。自分が作ったブランチ名に * がついていればOKです。
Step 3 / 5

✏️ VS Codeで作業する

ブランチを作ったら、VS Codeでスクリプトを編集します。Rojoを起動することでRoblox Studioにリアルタイムで反映されます。

Rojoサーバーを起動する

ターミナルで以下を実行：

rojo serve

Serving on port 34872 のようなメッセージが出たらOK。

Roblox Studioと接続する

  • 1
    Roblox Studioを開く
  • 2
    プラグインバーの Rojo アイコンをクリック → Connect
  • 3
    VS Codeでスクリプトを編集すると、Studioにリアルタイムで反映される
💡 作業が終わったら Rojoは Ctrl+C（Macも同じ）でターミナルから停止できます。次のステップでコミット作業を行います。
Step 4 / 5

☁️ 変更をコミット＆プッシュする

💡 「コミット」「プッシュ」って何？ コミットとは、変更内容に名前をつけて記録することです（セーブポイントのイメージ）。
プッシュとは、そのコミットをGitHubのオンライン上に送ることです。

① 変更を確認する

# 変更されたファイルの一覧を確認
git status

② 変更をステージングする

# 変更したファイルをすべて追加
git add .

③ コミットする

# 変更内容がわかるメッセージをつけて記録する
git commit -m "スコアボードを追加"
ℹ️ コミットメッセージについて " " の中は何をしたかを日本語で書いてOKです。後から見たときにわかる内容にしましょう。

④ GitHubにプッシュする

# 自分のブランチをGitHubに送る（ブランチ名は自分のものに変更）
git push origin feature/ブランチ名
Step 5 / 5

🔀 Pull Request（PR）を作成する

💡 「Pull Request（PR）」って何？ 自分のブランチの変更を main に取り込んでもらうための「申請」です。管理者が内容を確認・承認してからmainに反映されます。直接mainを書き換えるのではなく、必ずこのPRを通すことでミスや事故を防ぎます。
  • 1
    ブラウザで github.com/VR-SKATE/リポジトリ名 を開く
  • 2
    画面上部に緑色のバナーで Compare & pull request ボタンが表示されるのでクリック
    （表示されない場合は Pull requests タブ → New pull request）
  • 3
    タイトルに作業内容を入力（例：「スコアボードを追加」）
    必要に応じてコメント欄に詳細を記入する
  • 4
    Create pull request をクリック
  • 5
    Discordで管理者にPRを作成したことを連絡する
  • 6
    管理者が承認・mergeしたら完了 🎉
    次の作業はStep 1からやり直します
⚠️ コンフリクトが発生したら 他のメンバーが同じファイルを編集していた場合、競合（コンフリクト）が起きることがあります。無理に解決しようとせず、管理者に相談してください。
🎉
お疲れさまでした！

PRが承認されたら、この作業フローは完了です。
次の作業はまた Step 1 からスタートします。

📋 毎回の作業チェックリスト

  • 1
    git checkout main → git pull origin main
  • 2
    git checkout -b feature/ブランチ名（新しいブランチを作る）
  • 3
    VS Code + Rojoで作業する
  • 4
    git add . → git commit -m "内容" → git push origin feature/ブランチ名
  • 5
    GitHubでPull Requestを作成 → 管理者に連絡
