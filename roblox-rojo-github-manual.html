📋 GitHubアカウントはありますか？

Roblox + Rojoのコード管理にはGitHubを使います。すでにアカウントをお持ちですか？

ℹ️ GitHubとは コードをクラウド上で管理・共有するサービスです。チームでの共同作業に欠かせません。無料で使えます。
スクリプター向け ステップ 1-a

🔐 GitHubアカウントを作成する

以下の手順でGitHubアカウントを作成してください。

  • 1
    ブラウザで https://github.com を開く
  • 2
    右上の Sign up をクリック
  • 3
    メールアドレスを入力 → パスワードを設定 → ユーザー名（英数字）を決める
  • 4
    確認メールが届くので、メール内のリンクをクリックして認証を完了する
  • 5
    ログインできたら完了！自分のユーザー名を管理者に伝えてください（Organization招待に必要）
⚠️ ユーザー名について URLに表示されます（例: github.com/yourname）。チームに見せても問題ないものを選びましょう。

🔧 Gitはインストール済みですか？

GitはGitHubを使うために必要なツールです。VS Codeのターミナルで確認できます。

確認方法

VS Codeを開いて、Ctrl+`（MacはCmd+`）でターミナルを開き、以下を入力してください：

git --version

git version 2.x.x のように表示されれば✅ インストール済みです。
command not found などのエラーが出た場合は❌ 未インストールです。

スクリプター向け ステップ 4-a

⚙️ Gitをインストールする

Macには最初からGitが含まれている場合があります。ターミナルで以下を実行：

xcode-select --install

ポップアップが出たら「インストール」をクリック。これでGitが入ります。
または https://git-scm.com/download/mac からもインストールできます。

💡 Homebrewをお持ちの方 brew install git でも可能です。
  • 1
    https://git-scm.com/download/win を開く
  • 2
    64-bit Git for Windows Setup をダウンロード
  • 3
    インストーラーを実行。オプションはすべてデフォルトのままでOK
  • 4
    VS Codeを一度閉じて再起動し、ターミナルで git --version を確認
⚠️ Git for Windowsについて インストール時に「Git Bash」「Git GUI」なども一緒に入りますが、VS Codeのターミナルから使うので特に必要ありません。
スクリプター向け ステップ 5

🔴 Rojoはインストール済みですか？

RojoはRoblox Studioと外部エディター（VS Code）を繋ぐツールです。ターミナルで確認してみましょう。

💡 「Rojo」って何？ 通常、Robloxのスクリプトは Studio 上でしか編集できません。Rojoを使うと、VS Code で編集したファイルをリアルタイムで Studio に同期させることができます。これにより、GitでコードをチームBで管理しながらStudioで動作確認するという開発スタイルが実現します。
rojo --version
ℹ️ Rojoとは ファイルシステム上のLuaスクリプトをRoblox Studioに自動同期してくれるツール。VS Codeで編集した内容がリアルタイムでStudioに反映されます。
スクリプター向け ステップ 5-a

🔴 Rojoをインストールする

Rojoのインストールには Aftman（ツールバージョンマネージャー）を使う方法が推奨です。

① Aftmanをインストールする

Aftman Releases から最新の aftman-macos.zip をダウンロードして解凍し、ターミナルで：

./aftman self-install

その後、新しいターミナルセッションを開くか以下を実行：

source ~/.zshrc
# または ~/.bash_profile

Aftman Releases から最新の aftman-windows.zip をダウンロードして解凍。展開先のフォルダでPowerShellを開き：

.\aftman self-install

完了後、VS Codeを再起動してターミナルをリフレッシュ。

② Rojoをインストールする

プロジェクトフォルダ（後でクローンするリポジトリ）に aftman.toml があれば：

aftman install

なければ直接インストール：

aftman add rojo-rbx/rojo
⚠️ 別の方法：Ropeのダイレクトインストール プロジェクトで指定のバージョンを使う場合は管理者の指示に従ってください。
スクリプター向け ステップ 6

📦 リポジトリをクローンする

GitHubのプロジェクトファイル一式をあなたのPC上にコピーします。

💡 「リポジトリ」って何？ プロジェクトのファイルや変更履歴をまるごと保存しているフォルダのことです。GitHubの上にある「オンライン版」と、自分のPC上にある「ローカル版」の2つが存在します。「クローン」とは、オンライン版をそのままPC上にコピーしてくる作業のことです。

① リポジトリURLを取得する

  • 1
    ブラウザで https://github.com を開いてログインする
  • 2
    画面左上のアイコンをクリック → Your organizations を選択 → VR-SKATE をクリック
  • 3
    ページ上部の Repositories タブをクリックし、管理者から指定されたリポジトリ名をクリック
  • 4
    リポジトリのトップページが開いたら、緑色の <> Code ボタンをクリック
  • 5
    HTTPS タブを選択し、表示されたURL（例: https://github.com/VR-SKATE/repo.git）右横のコピーボタンをクリック

② VS Codeのターミナルでクローンする

VS Codeを起動し、Ctrl+`（Mac: Cmd+`）でターミナルを開く。

クローン先のフォルダに移動してから：

# 例: デスクトップに移動する場合
cd ~/Desktop

# リポジトリをクローン（URLは自分のものに変更）
git clone https://github.com/OrgName/repo.git
ℹ️ URLの "repo" について repo の部分はリポジトリ名です。GitHubのリポジトリページで確認できます。クローン後、同じ名前のフォルダがPC上に作成されます。

③ フォルダをVS Codeで開く

# "repo" の部分は実際のリポジトリ名に変更する
cd repo
code .

または VS Code のメニュー → File → Open Folder でクローンしたフォルダを選択。

✅ 完了の確認 VS Codeの左サイドバーにプロジェクトのファイル一覧が表示されていればOKです。

    または RojoのRobloxプラグインページ から「Get」でインストール

ℹ️ 既にプラグインが入っている場合 このステップはスキップ可能です。プラグインバーにRojoのアイコンがあればOK。

🚀 Rojoサーバーを起動して
Roblox Studioに接続する

① VS Codeのターミナルでサーバーを起動

VS Codeのターミナルで、プロジェクトのルートフォルダに移動してから：

💡 プロジェクトファイルについて プロジェクトに default.project.json がある場合はそのまま rojo serve でOK。
別名のファイルを使う場合: rojo serve myproject.project.json

② Roblox Studioから接続する

  • 1
    Roblox Studioを開き（Playtest用のPlaceファイルを開く）
  • 2
    プラグインバーの Rojo アイコンをクリック
  • 3
    Connect ボタンをクリック → サーバーに接続される
  • 4
    VS Codeでスクリプトを編集すると、Studioにリアルタイムで反映されます 🎉

🔄 日常の作業フロー

セットアップ完了後の基本的なGitワークフローです。毎回の作業でこの流れを繰り返します。

💡 「ブランチ」って何？ ブランチとは、プロジェクトのコードを「コピーして別の場所で作業する」イメージです。
みんなが共有している main（本番コード）を直接触らず、自分専用の作業スペース（ブランチ）を作ってそこで編集します。作業が完成したら、レビューを経て main に反映します。

こうすることで、未完成のコードが本番に混入したり、他のメンバーの作業を上書きしてしまうリスクを防げます。
ℹ️ ブランチ運用の流れ main ブランチは常に安定した状態を保ちます。作業は必ず自分のブランチで行い、完成したらPull Request（PR）を作成してレビュー後にmainへ反映します。

🌅 作業開始時（毎回）

# 最新のmainを取得する（必ず毎回やること！）
git checkout main
git pull origin main

🌿 作業ブランチを作る

💡 ブランチの名前は自由でOK ブランチは作業のたびに毎回新しく作ります。名前は何でも構いませんが、後で見たときに何の作業かわかる名前にすると管理しやすいです。
例：feature/enemy-ai（敵AIを追加）、fix/jump-bug（ジャンプのバグ修正）など
mergeされたブランチは役目が終わります。次の作業では、また新しいブランチを作り直します。
# 作業内容がわかる名前でブランチを作成して移動
# 例: feature/enemy-ai、fix/jump-bug など
git checkout -b feature/ブランチ名

✏️ 作業中

VS Codeでスクリプトを編集 → rojo serve でStudioに反映しながら作業

☁️ 作業後（変更をプッシュ）

# 変更したファイルを確認
git status

# 変更をステージング
git add .

# コミット（変更内容を簡潔に書く）
git commit -m "◯◯機能を追加"

# 自分のブランチをGitHubにプッシュ
git push origin feature/ブランチ名

🔀 Pull Request（PR）を作成する

💡 「Pull Request（PR）」って何？ 自分のブランチの変更内容を main に取り込んでもらうための「申請」です。GitHubのウェブ上で行い、管理者や他のメンバーがコードを確認・承認してからmainに反映されます。直接mainを書き換えるのではなく、必ずこのPRを通すことでミスや事故を防ぎます。
  • 1
    GitHubのリポジトリページを開く → 黄色いバナーの Compare & pull request をクリック（またはPull requestsタブ → New pull request）
  • 2
    タイトルと変更内容の説明を書いて Create pull request をクリック
  • 3
    管理者またはレビュー担当者にレビューを依頼する（Discordで連絡）
  • 4
    承認されたら Merge pull request → mainに反映される 🎉
⚠️ コンフリクトが発生したら 同じファイルを別のメンバーが編集していた場合、競合（コンフリクト）が起きることがあります。無理に解決しようとせず、管理者またはチームメンバーに相談してください。

🗂️ Roblox Studioでのアセット管理

Roblox StudioのExplorerを使って、Scene上のオブジェクトをフォルダで整理します。

Explorerでのフォルダ整理

  • 1
    Studio上部メニュー → View → Explorer を表示（ショートカット: Alt+X）
  • 2
    整理の基本構造：
    Workspace/
      📁 Map/ # 地形・建物など
      📁 Props/ # 小道具・アイテム
    ReplicatedStorage/
      📁 Assets/ # 共有アセット
    ServerStorage/
      📁 Templates/ # テンプレートモデル
  • 3
    Explorerで右クリック → Insert Object → Folder でフォルダを作成
💡 命名ルール（チームで統一推奨） フォルダやモデル名はわかりやすい英語で統一すると管理しやすくなります。例: NPC_Enemy_Goblin
アセット制作者向け ステップ 4

⬆️ アセットをアップロードする

「アセット管理」ウィンドウの バルクインポート を使って、ファイルをRobloxにアップロードします。

① アセット管理ウィンドウを開く

  • 1
    Studioの上部メニュー → ウィンドウ → アセット管理 をクリック
  • 2
    アセット管理ウィンドウが開く。カテゴリが表示される（画像・メッシュ・オーディオ・モデルなど）

② バルクインポートでアップロード

  • 1
    アセット管理ウィンドウ上部の ↑（アップロードボタン） をクリック → バルクインポートが起動する
  • 2
    アップロードしたいファイルを選択（複数同時選択OK）
    画像: .png / .jpg　メッシュ: .fbx / .obj　オーディオ: .mp3 / .ogg
  • 3
    開く をクリックするとアップロードが始まる
  • 4
    アップロード完了後、対応カテゴリフォルダ（例：画像 → 「画像」フォルダ）に自動的に追加される
⚠️ 画像のモデレーションについて アップロードした画像はRobloxの審査が入ります。承認まで数分〜数時間かかる場合があり、その間は使用できません。

📂 アセット管理ウィンドウを活用する

アセット管理ウィンドウでは、アップロード済みのアセットの確認・整理・IDのコピーができます。

アセット管理ウィンドウを開く

Studioの上部メニュー → ウィンドウ → アセット管理

カテゴリフォルダの構成

アセット管理にはアップロード済みのアセットが種類別に格納されています：

📁 プレース
📁 画像
📁 メッシュ
📁 パッケージ
📁 オーディオ
📁 モデル

主な使い方

  • 🔍
    確認：各カテゴリフォルダを開くとアップロード済みアセットが一覧表示される
  • 📋
    Asset IDのコピー：アセットを右クリック → アセットIDをコピー → スクリプターに共有できる
  • 📁
    フォルダで整理：カテゴリ内で右クリック → フォルダの追加 でサブフォルダを作成できる
  • ⬆️
    バルクインポート：上部の ↑（アップロードボタン） をクリックしてさらに追加できる
✅ チーム共有のポイント グループのPlaceからアップロードしたアセットは、同じグループのメンバーのToolbox（グループタブ）からも利用できます。
